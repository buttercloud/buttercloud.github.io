Getting Started

Welcome to the Upzaar GraphQL API documentation! Upzaar is a powerful suite of APIs designed to help businesses increase customer engagement, drive customer loyalty, and encourage repeat purchases.

Our GraphQL API allows you to interact with the Upzaar platform in a flexible and efficient way, giving you access to a comprehensive set of endpoints to build personalized loyalty experiences for your customers and B2B clients.

With Upzaar GraphQL API, you can easily fetch the data you need and utilize advanced features like nested queries. In this documentation, you'll find all the information you need to get started with our GraphQL API, including detailed guides, code samples, and troubleshooting tips. So, let's dive in and start building a personalized customer experience with Upzaar.

API Key​

Our GraphQL and REST APIs both require basic authentication with an API key.

Every Upzaar organization comes with an API key. Your organization may include multiple brands. If you require more API keys and environments, please contact us.

You can find your API keys on the Organization Settings page of your admin panel.

API Authentication​

Your GraphQL API requests are authenticated using an API key. An API key is a unique identifier that is generated by the server and passed to the client. The client includes the API key in the headers of every API request, and the server uses the key to authenticate the request. To authenticate your requests, pass the API key as a header. Example:

ApiKey <your-api-key>

Configuring your Tiers, Rewards and Brands​

The first step in setting up your account is to contact us and request a new organization account. Once created, you'll be able to sign in to your administration panel and begin setting up your loyalty offering.

To start out, you'll need to set up the following:

  1. Add one or more brands. These are B2B clients - multiple brands are supported - but you may choose to only have one.
  2. Define your rewards for each brand
  3. Define membership tiers for each brand (optional)
  4. Configure your loyalty settings like points per visit, welcome points, and point expiry.

Terminology​

It will be helpful to get a handle on the naming of a few objects in Upzaar's system.

BrandGroup: This is your organization.

Brand: This is a specific B2B brand under your organization. You can set up as many brands as you need - most requests require a brandUid for loyalty operations.

Customer: A specific end-user of your loyalty program, usually a customer.

Reward: Rewards belong to brands, and have a set amount of points that decrease a customer's point balance once redeemed.

Tier: A tier is a membership level that's defined for a brand. e.g Bronze, Silver, Gold members. Each tier has a minimum and maximum point value that determines if users fall into this specific tier.
